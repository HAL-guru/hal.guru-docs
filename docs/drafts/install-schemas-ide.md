## For JetBrains IDEs (IntelliJ IDEA, PyCharm, WebStorm, GoLand, RubyMine)

Associate the hal.guru schema with your YAML files:

1) Open Settings (Windows/Linux: File > Settings, macOS: IntelliJ IDEA > Settings or Preferences).
2) Navigate to Languages & Frameworks > Schemas and DTDs > JSON Schema.
3) Click the + icon to add a new schema mapping.
4) Set:
   - Name: hal.guru
   - Schema file: point to the local schema file installed/generated by halguru (for example, the path printed by the CLI) or a schema URL your team provides.
5) Under Mappings, add file patterns for your configuration files, for example:
   - hal.yaml, hal.yml
   - config/hal/*.yml
6) Apply and OK. Open a mapped file to see validation and completion.

Notes:
- If you use multiple modules or repos, add additional mappings per project.
- JetBrains IDEs cache schemas; if updates don’t appear, click “Reset” in the schema settings or restart the IDE.

## For Sublime Text (with LSP + YAML language server)

Prerequisites:
- Install “Package Control”.
- Install “LSP” and “LSP-yaml” packages.

Configure schema association:

1) Open Preferences > Package Settings > LSP-yaml > Settings.
2) In your user settings, add a mapping using yaml-language-server syntax:
```json
{
"settings": {
"yaml.schemas": {
"file:///absolute/path/to/your/halguru.schema.json": [
"hal.yaml",
"config/hal/*.yml"
]
}
}
}
```
Notes:
- Use a file:// URL for local files on most clients (“file:///path/to/file.json”).
- Adjust glob patterns to match your repository layout.

## Neovim (nvim-lspconfig) and Vim (coc.nvim)

Neovim (Lua, nvim-lspconfig + yaml-language-server):
```lua
require("lspconfig").yamlls.setup({
settings = {
yaml = {
schemas = {
["file:///absolute/path/to/halguru.schema.json"] = {
"hal.yaml",
"config/hal/*.yml",
},
},
validate = true,
hover = true,
completion = true,
},
},
})
```
Vim/Neovim with coc.nvim (coc-settings.json):
```json
{
"yaml.schemas": {
"file:///absolute/path/to/halguru.schema.json": [
"hal.yaml",
"config/hal/*.yml"
]
}
}
```
Notes:
- Ensure yaml-language-server is installed in your environment.
- Replace paths and glob patterns according to your project structure.

## Emacs (lsp-mode + yaml-language-server)

Prerequisites:
- Install yaml-mode and lsp-mode.
- Ensure yaml-language-server is available in PATH.

Add the schema mapping (init.el or equivalent):
```elisp
(use-package yaml-mode :ensure t)
(use-package lsp-mode :ensure t)

(with-eval-after-load 'lsp-mode
(lsp-register-custom-settings
'(("yaml.schemas"
(ht
("file:///absolute/path/to/halguru.schema.json"
["hal.yaml" "config/hal/*.yml"]))
t))))
```
Open a YAML file and run M-x lsp to activate validation and completion.

### For Eclipse (Wild Web Developer)

1) Install the “Wild Web Developer” plugin if it’s not already available.
2) Open Window > Preferences.
3) Navigate to Languages > YAML > Schemas (or YAML > Validation, depending on version).
4) Add a new schema:
   - Location: a schema URL or a local file (the schema installed/generated by halguru).
   - Patterns: hal.yaml, config/hal/*.yml (adjust to your project).
5) Apply and restart if prompted.

Notes:
- Different Eclipse versions place YAML settings under slightly different preference paths; search for “YAML” in Preferences if needed.

## Generic YAML language server clients

Most LSP-based editors accept the same setting:
```json
{
"yaml.schemas": {
"file:///absolute/path/to/halguru.schema.json": [
"hal.yaml",
"config/hal/*.yml"
]
}
}
```
Where:
- The key is a schema URL or file URI.
- The value is one or more glob patterns pointing to your configuration files.

Tip:
- If you installed the schema using the halguru CLI, use the exact path or URL it prints, then map it to your project’s YAML files.
